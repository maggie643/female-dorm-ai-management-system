# university-dorm-ai-management-system
高校宿舍智能AI管理系统

## 项目场景
针对高校宿舍传统线下报修、人工登记管理效率低下的问题，基于魔搭开源大模型开发AI自动化管理平台，覆盖全校所有宿舍的智能化运维管理。

## 核心AI能力
1. **智能故障分类**：基于Qwen大模型自动识别故障类型与紧急等级（P0最高危，P2普通）
2. **工单自动分配**：根据故障类型智能匹配维修人员技能，实现自动派单
3. **数据可视化分析**：多维度统计报表，支持饼图、折线图、柱状图等可视化展示
4. **超时预警机制**：实时监控工单进度，超时工单自动提醒
5. **重复报修检测**：智能识别同一宿舍近期重复报修，避免资源浪费
6. **服务评价体系**：工单完成后支持满意度评分与反馈，形成服务闭环

## 功能模块

### 📝 报修提交
- 在线填写报修信息（楼栋、房间号、故障描述）
- AI智能识别故障类型与紧急等级
- 自动生成工单编号
- 重复报修检测与预警提示

### 📋 工单管理
- 工单列表实时刷新
- 按状态筛选（待分配、处理中、已完成）
- 按楼栋/房间号搜索工单
- 超时工单预警
- 维修人员状态管理
- 工单完成标记
- 服务满意度评价（1-5星评分+文字反馈）

### 📊 数据分析
- 工单统计概览（总数、完成率、处理中、待分配、平均满意度）
- 工单状态分布饼图
- 紧急等级分布柱状图
- 近7日报修趋势折线图
- 楼栋报修排行柱状图
- 故障类型分布柱状图

## 技术架构
```
├── app.py              # Gradio Web界面（角色权限、报修、工单管理、数据分析）
├── ai_engine.py        # AI引擎模块（故障识别，4bit/8bit/全量三级回退）
├── workflow.py         # 工单流程模块（提交、派单、完成、评价、搜索）
├── db.py               # 数据库操作模块（SQLite，含评价字段和搜索功能）
└── dorm_management.db  # SQLite数据库
```

## 数据库设计

### 表结构
| 表名 | 说明 |
|------|------|
| fault_types | 故障类型字典表（类型名称、优先级、处理时限、技能要求） |
| repair_staff | 维修人员表（姓名、电话、技能、状态、当前工单） |
| work_orders | 工单表（工单编号、楼栋、房间、故障描述、状态、评价等） |

### 字段说明
**work_orders表新增字段：**
| 字段名 | 类型 | 说明 |
|--------|------|------|
| rating | INTEGER | 满意度评分（1-5星） |
| feedback | TEXT | 用户评价反馈 |
