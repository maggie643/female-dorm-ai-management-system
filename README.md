# university-dorm-ai-management-system
# 🤖 高校宿舍AI运营管理平台

## 🌟 赛事主题
**AI + 运营 / 「运营人，造自己的工具」**

运营工作里，总有一些重复、耗时又让人抓狂的事情。如果有一个工具能帮运营少加一天班，会是什么？

本项目就是这样一款提效工具——用AI把宿舍运营管理的想法变成一款真正可运行、可体验、可推广的智能运营平台。

---

## 🎯 项目定位

针对高校宿舍传统线下报修、人工登记管理效率低下的问题，基于魔搭开源大模型开发AI自动化管理平台，覆盖全校所有宿舍及港澳高校的智能化运维管理。

**核心价值：让宿舍运营更高效，让运营人少加班！**

---

## 🚀 核心AI能力

| 能力模块 | 功能描述 |
|----------|----------|
| 🤖 **智能故障分类** | 基于Qwen大模型自动识别故障类型与紧急等级（P0最高危，P2普通） |
| 📋 **工单自动分配** | 根据故障类型智能匹配维修人员技能，实现自动派单 |
| 📊 **数据可视化分析** | 多维度统计报表，支持饼图、折线图、柱状图等可视化展示 |
| ⚠️ **超时预警机制** | 实时监控工单进度，超时工单自动提醒 |
| 🔍 **重复报修检测** | 智能识别同一宿舍近期重复报修，避免资源浪费 |
| ⭐ **服务评价体系** | 工单完成后支持满意度评分与反馈，形成服务闭环 |
| 💬 **AI智能助手** | 基于知识库的问答机器人，自动回答宿舍管理规定、报修流程等问题 |
| 📝 **周报自动生成** | 一键生成专业运营周报，包含工作概览、关键指标、改进建议 |
| 🎯 **运营仪表盘** | 数据驾驶舱，让管理者一眼看懂运营数据 |

---

## 📱 功能模块

### 📝 报修提交
- 在线填写报修信息（高校、楼栋、房间号、故障描述）
- AI智能识别故障类型与紧急等级
- 自动生成工单编号
- 重复报修检测与预警提示
- **学生端反馈**：显示维修人员姓名和联系电话
- **老师端反馈**：显示学校负责人、联系电话、地址

### 📋 工单管理
- 工单列表实时刷新
- 按状态筛选（待分配、处理中、已完成）
- 按高校/楼栋/房间号搜索工单
- 超时工单预警
- 维修人员状态管理
- 工单完成标记
- 服务满意度评价（1-5星评分+文字反馈）
- **CSV数据导出**：方便老师人工整理数据

### 📊 数据分析
- 工单统计概览（总数、完成率、处理中、待分配、平均满意度）
- 工单状态分布饼图
- 紧急等级分布柱状图
- 近7日报修趋势折线图
- 楼栋报修排行柱状图
- 故障类型分布柱状图
- 高校报修统计

### 🤖 AI智能助手
- **智能问答**：基于知识库回答宿舍管理规定、报修流程、安全须知等问题
- **周报生成**：一键生成专业运营周报

### 📊 运营仪表盘
- 总工单数、完成率、平均满意度、待分配数实时展示
- 工单状态分布、紧急等级分布可视化
- 近7日报修趋势图
- 楼栋报修排行榜

## 🎉 项目亮点

1. **纯自研轻量化AI系统**：基于魔搭Qwen-1.8B-Chat模型，支持4bit/8bit/全量三级回退加载
2. **覆盖港澳高校**：支持香港、澳门高校的宿舍管理适配
3. **AI智能问答机器人**：学生咨询调宿、报修、规章制度自动答复，现有商用产品极少内置
4. **运营周报自动生成**：一键生成专业运营周报，节省运营人员时间
5. **运营数据驾驶舱**：让管理者一眼看懂运营数据
6. **开源可迭代**：GitHub开源项目，可灵活迭代、校内免费试点，后续可向周边院校轻量化推广

## 📝 赛事价值

本项目完美契合「AI + 运营」赛事主题，解决了宿舍运营中的以下痛点：

- **重复性工作**：AI自动分类故障、自动派单，减少人工操作
- **数据整理耗时**：一键导出CSV、自动生成周报，节省数据整理时间
- **信息咨询频繁**：AI智能助手7×24小时回答学生问题，减轻宿管负担
- **数据不透明**：运营仪表盘实时展示关键指标，让管理决策更精准
