# university-dorm-ai-management-system
高校宿舍智能AI管理系统

## 项目场景
针对高校宿舍传统线下报修、人工登记管理效率低下的问题，基于魔搭开源大模型开发AI自动化管理平台，覆盖全校所有宿舍的智能化运维管理。

## 核心AI能力
1. **智能故障分类**：基于Qwen大模型自动识别故障类型与紧急等级（P0最高危，P2普通）
2. **工单自动分配**：根据故障类型智能匹配维修人员技能，实现自动派单
3. **数据可视化分析**：多维度统计报表，支持饼图、折线图等可视化展示
4. **超时预警机制**：实时监控工单进度，超时工单自动提醒

## 功能模块

### 📝 报修提交
- 在线填写报修信息（楼栋、房间号、故障描述）
- AI智能识别故障类型与紧急等级
- 自动生成工单编号

### 📋 工单管理
- 工单列表实时刷新
- 按状态筛选（待分配、处理中、已完成）
- 超时工单预警
- 维修人员状态管理
- 工单完成标记

### 📊 数据分析
- 工单统计概览（总数、完成率、处理中、待分配）
- 工单状态分布饼图
- 紧急等级分布柱状图
- 近7日报修趋势折线图

## 技术架构
```
├── app.py              # Gradio Web界面
├── ai_engine.py        # AI引擎模块（故障识别）
├── workflow.py         # 工单流程模块
├── db.py               # 数据库操作模块
└── dorm_management.db  # SQLite数据库
```

## 数据库设计

### 表结构
| 表名 | 说明 |
|------|------|
| fault_types | 故障类型字典表（类型名称、优先级、处理时限、技能要求） |
| repair_staff | 维修人员表（姓名、电话、技能、状态） |
| work_orders | 工单表（工单编号、楼栋、房间、故障描述、状态等） |

### 工单状态流转
```
待分配(pending) → 处理中(processing) → 已完成(completed)
```

## 安装与运行

### 环境要求
- Python 3.8+
- PyTorch
- 建议GPU环境（模型推理加速）

### 安装依赖
```bash
pip install -r requirements.txt
```

### 启动应用
```bash
python app.py
```

### 访问地址
打开浏览器访问 `http://localhost:7860`
